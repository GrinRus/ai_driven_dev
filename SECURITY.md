# Security Policy

## Supported versions

Security fixes are provided for the latest release branch and the current `main` branch.

## Reporting a vulnerability

For sensitive vulnerabilities, use GitHub private vulnerability reporting:
- [Create private advisory](https://github.com/GrinRus/ai_driven_dev/security/advisories/new)

If private reporting is unavailable, open a minimal issue with title prefix `[SECURITY]` and avoid posting exploit details.

## Response SLA

- Initial triage response: within 3 business days.
- Severity assessment and mitigation plan: within 7 business days.
- Public disclosure timeline: coordinated after a fix is available.

## Disclosure policy

- Do not publish proof-of-concept details before maintainers confirm remediation.
- Credits are added in release notes after coordinated disclosure.
